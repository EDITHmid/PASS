Five-Factor Credibility Model

Statistically grounded. Transparent. Configurable per school.

Δt Consistency
Weight: 25%

Tracks submission timing relative to deadlines. A student submitting 24h early gets full marks; submissions near deadlines or late indicate drift.

Variance Stability
Weight: 10%

Rolling variance of Δt flags erratic submission patterns as burnout precursors — even when grades are still passing.

Completion Rate
Weight: 10%

Fraction of assignments submitted. Combined with hysteresis filtering that requires 3 consecutive declines before alerting.

Attendance
Weight: 25%

Class attendance % mapped to 0–100. Students below 75% attendance are flagged as high-risk regardless of other scores.

Exam Performance
Weight: 30%

Best 2 of 3 mid-term scores averaged and scaled to 0–100. Ensures academic mastery is measured alongside behavioral signals.

Configurable by Principal: All five weights are adjustable from the admin dashboard — customize for your school's priorities.

ML Black Box
Core Algorithm: Hysteresis Filtering

Inspired by control theory, the hysteresis filter requires 3 consecutive declining submissions before triggering an alert, and 2 consecutive improvements to auto-resolve. This eliminates 60%+ of false positives compared to single-threshold systems.

Architecture

Three-tier design: Data Ingestion (Pandas CSV parser) → Analytical Engine (Flask + NumPy metrics, hysteresis, credibility scorer) → Application Layer (Bootstrap 5 dashboards with Chart.js). Deployable on any Python 3.10+ host.
